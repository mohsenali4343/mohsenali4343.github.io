MnMs.io Documentation

MnMs (Monitoring and Management System) is a secure, no-login-required DevOps platform that allows teams to monitor and manage production machines while preserving strict separation of access among stakeholders. MnMs enforces Zero-Trust security principles and supports safe, script-based management.

Stakeholder Roles

Business Owners

Own the data; control access and security credentials.

DevOps

Operate and manage the system without direct data access.

App Developers

Build and maintain the application.

Infrastructure Providers

Provision and manage physical/root-level machine access.

Architecture

MnMs uses a lightweight "Trusted Agent" that runs on all production machines.

Components

  • •
    Trusted AgentInstalled on production machines with root access; open-source.
  • •
    Monitoring ClientCollects system data and sends it to the Monitoring Server.
  • •
    Monitoring ServerCloud-based server aggregating monitoring data.
  • •
    Management AgentExecutes safe, pre-defined scripts for operations tasks.
  • •
    ManagerCloud-side controller for Management Agents.

Access Model

  • •
    DevOps interact with MnMs via a GUI and an API.
  • •
    No direct login to production machines is required or allowed.
  • •
    Only enumerated, validated scripts can be executed.

Deployment Lifecycle

Infrastructure Provider

  • •Creates machine images with MnMs Trusted Agent and secrets.
  • •Instantiates or provides machines to DevOps.

App Developers

  • •Deliver production app builds and continuous updates.

Business Owners

  • •Customize the app and manage secrets.
  • •Own all data collected.

DevOps

  • •Operate and manage machines exclusively through MnMs servers.

Key Features

OS-Level Security Enforcement

  • •Unique OS-level user accounts for every process.
  • •Enforces Least Access Privilege.

Peer Authentication for Databases

  • •Uses OS-level credentials for DB access.
  • •Eliminates need for hardcoded passwords or .env secrets.

Multi-Tenant & Microservice Support

  • •Processes and databases are sandboxed by OS user.
  • •Allows multiple apps or tenants on the same machine securely.

Full Observability

  • •Instrumentation throughout request flows.
  • •OpenTelemetry support.

Safe-Script Framework

  • •Pre-validated and sanitized
  • •Logged in sudo logs
  • •Configurable for append-only remote logs
  • •Extensible by Business Owners

Zero-Trust with mTLS Authentication

  • •mTLS between all production elements.
  • •Two-Factor Authentication using DevOps and Business Owner credentials.
  • •Prevents full access compromise from either side.

Installation

Using Package Manager

curl -sSL https://get.mnms.io | bash

This will install the latest stable version of MnMs on your system.

Manual Installation

1. Download the Package

wget https://download.mnms.io/latest/mnms-agent.tar.gz

2. Extract the Package

tar -xzf mnms-agent.tar.gz

3. Run the Installer

cd mnms-agent && sudo ./install.sh

Verification

After installation, verify that the MnMs agent is running correctly:

sudo systemctl status mnms-agent
